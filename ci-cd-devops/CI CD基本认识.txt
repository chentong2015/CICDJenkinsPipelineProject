Continue Integration & Continue Deployment/Delivery
1. CI要求开发者集成(commit)代码到指定的共享仓库(share codebase)
   开发者能够访问到最新的集成成功的代码
2. 通过频繁的集成(build process)能够尽快的发现代码中的问题
   及时反馈及时修复，降低风险，减少软件维护的成本 !!
3. CI能够可能保证提交的代码不会其它项目的组的模块造成问题
4. CI能够自动生成可部署的软件，可用于交付的代码

Principles of Continuous Integration:
Code Dev -> Source Control -> Build -> Test(Unit Test, UI Test, Security Test)

The Benefits of Continuous Integration:
- Higher Quality
- Faster Delivery
- Lower Costs
- More Flexibility
